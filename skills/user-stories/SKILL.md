# User Stories

## Goal

Express requirements from the user's point of view so design, engineering, and QA understand intent.

## Workflow

1. Identify actor, goal, context, and expected value.
2. Write stories using `As a <user>, I want <capability>, so that <outcome>`.
3. Add scenario notes for entry point, trigger, and completion.
4. Connect each story to measurable product outcomes where possible.

## Output

- User story list
- Scenario table
- Priority signal
- Linked requirement or metric

## Quality Bar

- Each story has a clear actor and outcome.
- Stories are not implementation tasks.
- Stories connect to PRD scope.
