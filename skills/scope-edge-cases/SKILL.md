# Scope and Edge Cases

## Goal

Prevent scope drift and missed review issues.

## Workflow

1. Separate in-scope, out-of-scope, and future-scope work.
2. Identify dependencies across design, engineering, data, legal, operations, and support.
3. Cover edge cases for empty states, errors, permissions, cancellation, payment, retries, rollback, and notifications.
4. Mark assumptions and unresolved decisions.

## Output

- Scope table
- Non-goals
- Edge case matrix
- Dependency list
- Risk notes

## Quality Bar

- Boundaries are explicit.
- Cross-functional dependencies are visible.
- Edge cases are specific enough for QA review.
