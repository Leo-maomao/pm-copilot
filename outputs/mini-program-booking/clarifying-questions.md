# Clarifying Questions

## Must Answer Before Launch

1. What authorization scope is required before booking?
2. Which contact fields are mandatory?
3. How long is a selected time slot held before expiration?
4. Can users cancel or reschedule in v1?
5. What operational system receives confirmed bookings?

## Can Proceed With Assumptions for Draft

1. Payment is out of scope.
2. Authorization happens before booking submission.
3. Slot availability is revalidated on submit.
4. The form uses minimal contact fields.
5. Booking failures use mapped error categories, not raw API payloads.

## Can Decide Later

1. Reminder strategy.
2. Cancellation and rescheduling policy.
3. Staff assignment display.
4. Calendar integration.
