# Final Package Contract

The final package summary must help reviewers understand what exists and what still needs discussion.

## Required Sections

- Executive summary
- Artifact index
- Key product decisions
- Metrics and tracking summary
- Prototype summary
- Review status
- Assumptions
- Open questions
- Risks
- Recommended review agenda
- Next actions

## Rules

- Link or reference each artifact by filename.
- Keep unresolved items visible.
- Do not mark the package as ready if critical issues remain.
