# Memory Model

PM Copilot uses file-based memory in v1. It does not require a database or cloud memory service.

## Long-Term Memory

Stored in `context/`.

- Product category and business model
- User segments and personas
- North star metric and guardrail metrics
- Tracking taxonomy and forbidden properties
- PRD style preferences
- Known competitors
- Prototype platform preferences

## Run Memory

Stored in `outputs/<run-id>/` when the agent creates a runtime trace.

- Raw request
- Clarifying questions
- User answers
- Assumptions
- Agent decisions
- Artifact paths
- Review result

## Runtime Trace

Generated by PM Orchestrator for each run when a persistent trace is useful.

```yaml
task_id:
request:
context_files:
agents:
skills:
tools:
decisions:
assumptions:
human_confirmations:
artifacts:
review_status:
```

## Update Rules

- Do not overwrite user product context during a task run.
- Suggest context updates separately when a task reveals reusable information.
- Keep task-specific assumptions out of long-term memory unless the user confirms they are reusable.
